# Тестирование мобильного приложения Яндекс Метро (Android)
 
> Функциональное и регрессионное тестирование мобильного приложения после рефакторинга кода для проверки готовности новой версии к релизу.
 
## Стек
 
Android Studio (эмулятор), Google Таблицы, mindmap
 
## Задача
 
Команда провела рефакторинг приложения Яндекс Метро на Android. Перед выпуском новой версии нужно было протестировать изменённую функциональность, провести регрессионное тестирование и подготовить отчёт для менеджера.
 
## Что я сделал
 
- Составил чек-лист на функциональное тестирование затронутых частей приложения
- Составил чек-лист на регрессионное тестирование с учётом мобильной специфики: функциональность, взаимодействующая с устройством, и обязательные проверки (например, тестирование обновления)
- Провёл тестирование на заданной конфигурации (Honor 8, Android 9.0 Pie, 1080×1920) на эмуляторе Android Studio
- Завёл баг-репорты и подготовил отчёт о тестировании для менеджера
## Результаты
 
- Проверок в чек-листах: 64
- Найдено дефектов: 9
- Вывод по готовности версии к релизу: _заполнить_
