# Регрессионное тестирование сервиса Mesto

> Регрессионное тестирование веб-приложения по готовым тест-кейсам с последующим ретестом ранее найденных дефектов.

## Стек

DevTools, Google Таблицы

## Задача

Протестировать веб-сервис Mesto по готовым тест-кейсам, проставить статусы проверок и оформить баг-репорты. Затем провести ретест багов, найденных другими тестировщиками, и убедиться, что разработчики их исправили.

## Что я сделал

- Изучил тест-кейсы для регрессионного тестирования
- Провёл тестирование на тестовом стенде, проставил статусы (PASSED / FAILED)
- Оформил баг-репорты: чёткий заголовок, приоритет, точные шаги воспроизведения, ожидаемый и фактический результат, скриншоты
- Выполнил ретест ранее заведённых дефектов: подтверждал исправления (статус «Закрыт») или переоткрывал баги (статус «Переоткрыт»)
- Фиксировал резолюцию в комментариях с указанием стенда, на котором проводилась проверка

## Результаты

- Пройдено тест-кейсов: _17_
- Найдено дефектов: _6_
- Проведён ретест: _15_ багов
