# Тест-дизайн с анализом требований

> Разработка тестовой документации на основе анализа требований к полю ввода и проверка функциональности.

## Стек

Техники тест-дизайна, Google Таблицы

## Задача

Проанализировать требования к фиче, выделить объект тестирования и разработать тест-кейсы с применением техник тест-дизайна, а затем выполнить тестирование полей «Часы», «Минуты», «Откуда», «Куда».

## Что я сделал

- Проанализировал требования к фиче и полям ввода, выделил объект тестирования
- Уточнял у эксперта недостающие и неоднозначные требования, чтобы исключить «серые зоны»
- Выделил классы эквивалентности (КЭ) и граничные значения (ГЗ)
- Описал тест-кейсы по заданному шаблону
- Протестировал поля «Часы», «Минуты», «Откуда», «Куда», зафиксировал результаты и оформил баг-репорты
- Протестировал расчет времени в пути и стоимость поездки

## Результаты

- Тест-кейсов: _69_
- Найдено дефектов: _13_
