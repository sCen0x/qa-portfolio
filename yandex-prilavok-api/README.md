# Тестирование API сервиса Яндекс.Прилавок

> Тестирование новой функциональности REST API интернет-магазина: работа с наборами, корзиной и расчётом курьерской доставки.

## Стек

Postman, Apidoc, Google Таблицы

## Задача

Разработчики добавили новую функциональность в API Яндекс.Прилавка и передали новую версию на тестирование. Нужно было покрыть тестами работу с наборами, корзиной и расчётом доставки (авторизацию проверять не требовалось).

## Что я сделал

- Проанализировал требования к бэкенду и изучил документацию API (Apidoc)
- Спроектировал чек-лист для покрытия переданной функциональности:
  - работа с наборами — `POST /api/v1/kits/{id}/products`
  - расчёт курьерской доставки — `POST /fast-delivery/.../calculate-delivery.xml`
  - корзина: получение `GET /api/v1/orders/{id}`, добавление `PUT /api/v1/orders/:id`, удаление `DELETE /api/v1/orders/:id`
- Протестировал методы GET, POST, PUT, DELETE через Postman: проверял статус-коды и тела ответов
- Завёл баг-репорты по найденным дефектам

## Результаты

- Проверок в чек-листе: _N_
- Найдено дефектов: _M_
- Краткий вывод по качеству API: _заполнить_
